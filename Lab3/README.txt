Файлы:
1. Library.cs (Библиотека классов + парсеры для xml и json)
2. MyService.cs (Windows-служба с классом для работы с конфигами)
3. configjson.json (Конфиг json)
4. configxml.xml (Конфиг xml)
   Нет воображения, чтобы придумать файлам нормальные названия)
   
При запуске службы:
1. В методе OnStart стоит логгер, который устанавливает опции для работы службы.
2. Создаётся объект класса(менеджер конфигов), который отвечает за парсинг.
3. Устанавливается нужный парсер(xml\json) в менеджер конфигов.
4. Достаём необходимый функционал в соответствии с типом конфига.
